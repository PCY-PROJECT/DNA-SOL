# 风控策略参考

## 硬性规则（不可绕过）

1. 单笔交易风险 ≤ 账户净值 2%
2. 总持仓风险暴露 ≤ 账户净值 10%
3. 未经用户明确授权，不得调用 order-execution MCP
4. order-execution MCP 未配置时，不得假装下单
5. 不得使用杠杆超过用户配置的上限（默认 1x）

## 风控检查清单

```
□ 仓位大小符合固定风险模型
□ 止损设置合理（非零，非极端）
□ 单笔风险 ≤ 账户 2%
□ 当前总风险暴露在限制内
□ market-data MCP 已配置且数据新鲜（≤5分钟）
□ account-read MCP 已配置且余额已确认
□ 用户已理解不保证盈利
```

## 触发否决的条件

以下任一条件触发否决，拒绝进入下单流程：

- 止损缺失
- 单笔风险 > 账户 2%
- market-data 数据超过 5 分钟
- account-read MCP 未配置
- order-execution MCP 未配置但用户要求真实下单
